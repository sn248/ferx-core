# NONMEM anchor — built-in absorption models (transit / inverse-Gaussian)

Reference NONMEM runs for the built-in absorption input-rate functions, to satisfy
the CLAUDE.md "compare with NONMEM output" rule:

| Anchors | ferx feature | NONMEM control stream | ferx model |
|---------|--------------|-----------------------|------------|
| **Savic transit** | `transit(n, mtt)` — PR [#343](https://github.com/FeRx-NLME/ferx-core/pull/343) | `savic_transit.ctl` | `transit_savic_fit.ferx` |
| **Freijer & Post IG** | `igd(mat, cv2)` — issue [#347](https://github.com/FeRx-NLME/ferx-core/issues/347) | `freijer_ig.ctl` | *(after #347 is implemented)* |

Both control streams run on **one** dataset, `transit_oral.csv`.

## The dataset

`transit_oral.csv` — a single-dose oral PK dataset, generated by
`simulate_transit_data.py` (pure stdlib, deterministic seed=7; re-run to reproduce
byte-for-byte). It is simulated from the **Savic transit model** with the same
truths as `examples/transit_savic.ferx`:

- 20 subjects, 100 mg oral dose into **CMT 1** at t=0, observations on **CMT 2**
  at 0.25–24 h (12 samples/subject, 240 obs).
- Truths: `TVCL=5`, `TVV=50`, `TVKA=1`, `TVMTT=1`, `TVN=3`; IIV on CL & V
  (ω²=0.09); proportional residual SD 0.15.
- Layout matches the 1-cpt built-ins: **depot = CMT 1, central = CMT 2**. (The
  repo's `data/transit_2cpt.csv` is *not* reused: it is the explicit 3-transit +
  **2-cpt** chain with observations on CMT 4, whose compartment layout doesn't map
  onto the 1-cpt Savic/IG built-ins.)

`DV` is the central **concentration** (mg/L). The dose feeds the absorption
function over time and is **not** added as a bolus — the ferx `transit()`
convention and the NONMEM `F1=0` + `PODO` trick (see notes).

## Run it

NONMEM (adjust to your install / PsN):

```bash
nmfe75 savic_transit.ctl savic_transit.lst
nmfe75 freijer_ig.ctl    freijer_ig.lst
# or:  execute savic_transit.ctl ;  execute freijer_ig.ctl
```

ferx — Savic transit (needs the #343 branch, where `transit()` exists):

```bash
cargo run --release -- nonmem_anchor/transit_savic_fit.ferx \
  --data nonmem_anchor/transit_oral.csv
# -> transit_savic_fit-fit.yaml (OFV + theta/omega/sigma + SEs)
```

ferx — IG (`igd()`): once #347 lands, add a `freijer_igd_fit.ferx` mirroring
`freijer_ig.ctl` (CL, V, MAT, CV2; `d/dt(central) = igd(mat=MAT, cv2=CV2) - CL/V*central`)
and fit it on the same data.

`results/ferx_savic_fit.yaml` (`transit_savic_fit.ferx`, `--features ci`, ferx

| ω²(CL) | 0.09 | 0.0480 | 0.0783 | +63% |
| ω²(V)  | 0.09 | 0.0431 | 0.0950 | +120% |
| σ² (prop) | 0.0225 | 0.0255 | 0.0253 | −0.6% |

**Verdict.** Structural model validated: all five fixed effects agree with
NONMEM within ~7% and both engines recover the simulation truths — the
`transit()` input-rate function + Lanczos `ln_gamma` are correct. Residual
error matches to <1%.

The ω² gap above (ferx +60–120% high) and the ~8-unit OFV offset are **an ODE
solver-tolerance artifact, not a FOCEI/marginal-likelihood issue.** The ferx
defaults (`ode_reltol=1e-4`, `ode_abstol=1e-6`) are far looser than NONMEM's
`TOL=9` (~1e-9); the resulting integration noise propagates into the FOCEI
gradient/Hessian and biases the variance components high. Re-fitting with
NONMEM-equivalent accuracy closes the gap entirely:

| Quantity | NONMEM | ferx (default 1e-4) | ferx (`ode_reltol/abstol=1e-9`, `inner_tol=1e-6`) |
|----------|--------|---------------------|---------------------------------------------------|
| OFV   | −1077.13 | −1069.07 (+8.05) | −1076.67 (**+0.46**) |
| TVCL  | 5.386 | 5.240 | 5.453 (+1.2%) |
| TVV   | 56.169 | 52.563 | 55.759 (−0.7%) |
| TVKA  | 0.952 | 1.018 | 0.950 (−0.2%) |
| TVMTT | 0.965 | 1.007 | 0.966 (+0.1%) |
| TVN   | 3.133 | 3.033 | 3.128 (−0.2%) |
| ω²(CL) | 0.0480 | 0.0783 | 0.0409 (−15%) |
| ω²(V)  | 0.0431 | 0.0950 | 0.0489 (+14%) |
| σ² (prop) | 0.0255 | 0.0253 | 0.0254 (−0.4%) |

With tight tolerances OFV agrees to **0.46**, thetas to **~1%**, and the ω²
bracket NONMEM at ±15% (n=20 sampling noise). Cost: ~5 min vs 67 s. Takeaway:
for variance-component accuracy on transit/stiff ODE models, set
`ode_reltol`/`ode_abstol` near `1e-9`; the loose defaults are fine for point
estimates of fixed effects but visibly inflate ω².

For Savic, also check parameter **recovery** vs the truths above (the data were
simulated from this model). For IG, recovery is not meaningful (mis-specified vs
the transit DGP); the test there is **NONMEM-igd ≈ ferx-igd**, not recovery.

## Implementation notes (why the streams look the way they do)

- **Lanczos `ln_gamma`.** `savic_transit.ctl` computes `ln Γ(N+1)` with the exact
  g=7/n=9 Lanczos coefficients from ferx `src/stats/special.rs::ln_gamma`, in
  `$PK` (N is constant over time). This removes the gamma special-function as a
  source of ferx-vs-NONMEM discrepancy — bare Stirling errs ~8% at N=1 and would
  bias the absorption peak / the TVN estimate.
- **Dose routing (`F1=0` + `PODO`).** `PODO` (last oral dose amount) is **not** a
  NONMEM reserved variable — it is captured in `$PK` with
  `IF(AMT.GT.0.AND.CMT.EQ.1) PODO=AMT` (likewise `TDOS=TIME`), and NONMEM carries
  it forward across records so `$DES` can read it on observation rows. `F1=0` then
  suppresses the depot bolus and `PODO` drives `R_in` — the NONMEM analogue of
  ferx delivering the whole dose as `R_in(tad)` and suppressing the instantaneous
  bolus (∫R_in dt = dose; no double counting). With bioavailability use `BIO*PODO`
  in `$DES` (`F1=0` removes the normal F path).
- **Time-after-dose `TAD = T - TDOS`.** `T` in `$DES` is absolute integration time;
  `TDOS` (captured in `$PK`) is the dose time, so `TAD = T - TDOS` is the
  time-after-dose (= `T` here, the dose being at t=0). For overlapping multi-dose
  data, `R_in` would additionally need superposing across doses (single dose here).
- **Concentration.** NONMEM tracks `A(2)` as an amount; `IPRED = A(2)/V` is the
  concentration, matching ferx's concentration-form central ODE. Estimates are
  identical to a `S2=V` scaling.

## Regenerating / variants

- Reproduce the dataset: `python3 simulate_transit_data.py`.
- **IG-truth dataset** (matched data for a clean IG anchor): change the input rate
  in `simulate_transit_data.py` to the inverse-Gaussian density
  `R_in = dose·sqrt(MAT/(2π·CV2·t³))·exp(-(t-MAT)²/(2·CV2·MAT·t))` driving central
  directly (drop the depot/KA step), pick `MAT≈2`, `CV2≈0.3`, and write a second
  CSV. Then `freijer_ig.ctl` recovers its truths.

These files are validation tooling run locally; they are **not** wired into CI
(NONMEM isn't available there).
