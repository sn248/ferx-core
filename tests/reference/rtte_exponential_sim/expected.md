# RTTE simulation — cross-tool anchor (Phase 3 Slice 3.3)

This directory is the **external** validation leg for ferx's RTTE **simulator**
(`simulate()` on a `type = rtte` model). The other, in-tree legs live in
`tests/tte_convergence.rs`:

- `rtte_forward_event_times_match_model_survival` — PIT/KS goodness-of-fit: the
  rescaled inter-event increments of the clock-forward sampler are `Exp(1)`
  (time-rescaling theorem), so the conditional draws reproduce `H(t)` exactly.
- `rtte_sampler_injects_correct_frailty_variance` — estimator-free method-of-moments:
  the simulated event counts imply `Var(log λ_i) = ω² = 0.09` (at high events/subject,
  where the read is tight).

## The cross-tool idea

ferx **simulated** the dataset `rtte_sim.csv` and then ferx **and NONMEM** both **fit**
it. An *independent* estimation engine (NONMEM) recovering the data-generating
parameters from ferx-simulated data is the external corroboration that the simulator is
correct: a biased sampler would move *both* engines off the data-generating values in
the same direction, and the two-engine agreement would still hold but land away from
the truth.

## Truth (data-generating)

Clock-forward (Andersen–Gill) constant-hazard RTTE with a shared log-rate frailty:

```
TVLAMBDA = 0.15   ω²(log rate) = 0.09   horizon = 20   N = 300 subjects
```

`rtte_sim.csv` (NONMEM format `ID,TIME,DV,EVID,CMT,MDV`) was generated by ferx's own
sampler:

```
cargo run --bin rtte_sim_anchor --features survival -- \
    tests/reference/rtte_exponential_sim/rtte_sim.csv
```

It has **999 events over 300 subjects (3.33/subject)**, one recurrent stream per
subject ending in an administrative censor (`DV = 0`) at `t = 20`.

## Fits of `rtte_sim.csv`

| Engine | Method | TVLAMBDA | ω²(log rate) | OFV |
|--------|--------|---------:|-------------:|----:|
| **NONMEM** | LAPLACE (telescoping-AG `$PRED`) | 0.15617 | 0.12801 | 5557.266 |
| **ferx**   | FOCEI                            | 0.15619 | 0.12794 | 5557.266 |

ferx FOCEI reproduces NONMEM LAPLACE to 4 significant figures and the OFV **to the
decimal** — the same equivalence the Slice 3.1 fit anchor established
(`../rtte_exponential`), now demonstrated on **ferx-simulated** rather than
R-simulated data.

Both engines land at ω² ≈ 0.128, above the data-generating 0.09. That is **not** a
simulator or estimator defect: this single N = 300 realization is over-dispersed (its
count-based method-of-moments ω² ≈ 0.125), and both engines faithfully fit the data in
front of them. The sampler's *true* injected variance is 0.09 — pinned estimator-free
at high events/subject by `rtte_sampler_injects_correct_frailty_variance` (MoM ω² =
0.0917 at ~31 events/subject).

## Reproducing NONMEM

```
docker run --rm -v "$PWD":/work -w /work nonmemdocker:V0.1 \
    /opt/NONMEM/nm760/run/nmfe76 nonmem.ctl nonmem.lst
```

The committed `nonmem.lst` / `nonmem.ext` are the outputs of that run. `nonmem.ctl` is
the Slice 3.1 telescoping-AG likelihood (`Y = SURV·λ` at events, `SURV` at the censor,
`TLAST` reset per subject), pointed at `rtte_sim.csv`. The ferx side is the committed
test `rtte_sim_anchor_ferx_matches_nonmem`.
