# Expected values: constant-hazard RTTE (repeated time-to-event)

Dataset: `rtte_exp.csv` — 100 subjects, constant per-subject hazard
`lambda_i = TVLAMBDA * exp(eta_i)` with data-generating `TVLAMBDA = 0.15`,
`omega^2(log rate) = 0.09`, administrative right-censoring at `t = 20`. Generated by
`simulate.R` (seed = 31) → **305 events, 100 censoring rows, 3.05 events/subject**.

Clock-forward (Andersen–Gill): the hazard is a function of absolute time (here constant)
and the cumulative hazard is **not** reset at events. The per-subject NLL is
`H(T) − Σ_k log h(t_k)`.

## Anchors

Two of the anchors below are **exact and license-free**; they are the primary
correctness checks. NONMEM LAPLACE is the cross-tool NLME reference.

### 1. Fixed-effects (n_eta = 0) — exact analytic MLE

Without frailty the clock-forward RTTE log-likelihood is
`log L = D·log(lambda) − lambda·Σ_i T_i`, whose maximiser is the pooled Poisson-process
rate `lambda_hat = D / Σ_i T_i = 305 / (100·20) = 0.15250`. A ferx fixed-effects fit
(`rtte_exp_fixed.ferx`, `method = focei`, no omega) must recover it exactly.

| Source | TVLAMBDA |
|---|---|
| Analytic MLE `D/Σt` | **0.15250** |
| Poisson GLM (`glm(n ~ 1, offset=log T)`, `count_equivalence.R`) | 0.15250 |
| **ferx fixed-effects** | **0.152500** (SE 0.00873, OFV 1757.16) |

### 2. Mixed-effects — exact via the count / Poisson-lognormal equivalence

For a constant hazard the RTTE likelihood factorises: the per-subject event **count**
`N_i ~ Poisson(lambda_i · T)` and the event times given `N_i` are uniform (independent of
`lambda`). So the `(lambda, omega^2)` MLE of the full RTTE data equals that of a
Poisson-lognormal GLMM on the counts — an **exact** mixed-effects reference with no
RTTE-specific software (`glmer`, `count_equivalence.R`). It is the MLE for *this*
realization, so ferx must match the GLMM, not the data-generating values (the frailty
`omega^2` is weakly identified at N=100 and this draw sits high).

| Source | TVLAMBDA | omega^2(log rate) | OFV |
|---|---|---|---|
| Poisson-LN GLMM (exact MLE) | 0.14062 | 0.16455 | — |
| **NONMEM** LAPLACE (`nonmem.ctl`) | **0.140618** (SE 0.01274) | **0.164496** (SE 0.07283) | **1748.493** |
| **ferx FOCEI** | **0.14062** (SE 0.01098) | **0.164714** (SE 0.07468) | **1748.493** |
| **ferx SAEM** (seed 31) | 0.14033 (SE 0.01101) | 0.169931 (SE 0.07532) | 1748.498 |
| data-generating | 0.15000 | 0.09000 | — |

ferx FOCEI reproduces **NONMEM LAPLACE to 4–5 significant figures on every parameter and
the OFV to the decimal** (1748.493), and both land on the exact Poisson-LN GLMM MLE. ferx
SAEM agrees within its stochastic tolerance.

## Notes

- **Laplace is unbiased here.** The Karlsson et al. (2009) −91…−96% `omega^2` bias for
  RTTE under Laplace is a *low-event-rate* phenomenon (event rate < 43%). This dataset is
  event-rich (every subject has events, mean 3.05), so FOCEI ≈ SAEM ≈ NONMEM ≈ GLMM. The
  ferx `method = saem`/`imp` recommendation warning still fires (it is a standing caution),
  but the estimates coincide. A sparser design would separate them.
- **OFV convention.** ferx and NONMEM (LIKELIHOOD) report `−2·Σ log L` without the
  Gaussian `N·log(2π)` constant (there are no normally-distributed observations), so the
  OFVs are directly comparable — hence the exact 1748.493 match.
- **NONMEM control stream.** Pure-TTE model (no `F_FLAG`, which is only for mixed
  continuous+likelihood data); `$EST … LIKELIHOOD`. Constant hazard needs no `$DES` — a
  `TLAST` reset in `$PRED` makes each row contribute the survival over its inter-event gap,
  which telescopes to the AG likelihood. Full output in `nonmem.lst` / `nonmem.ext`.

## Reproduce

```
Rscript simulate.R                 # regenerate rtte_exp.csv (seed 31)
Rscript count_equivalence.R        # exact fixed + mixed anchors (base R + lme4)
ferx rtte_exp_fixed.ferx --data rtte_exp.csv   # ferx fixed-effects  -> 0.15250
ferx rtte_exp.ferx       --data rtte_exp.csv   # ferx SAEM mixed fit
# NONMEM (Docker): nmfe76 nonmem.ctl nonmem.lst
```
